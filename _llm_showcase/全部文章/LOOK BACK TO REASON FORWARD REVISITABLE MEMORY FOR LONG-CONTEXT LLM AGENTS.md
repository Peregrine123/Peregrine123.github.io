## Abstract + Introduction
### 背景
- RAG方法
- Memorize while Reading 范式: 模型线性的扫描文档, 将当前分块与旧内村压缩为新内村, 这种方法能够处理无线上下文, 但是首先与线性处理逻辑
### 挑战
- 潜在证据的过早裁剪: Agent仅依据当前内存判断分块重要性, 但是多步推理所需的证据可能在阅读后期才显现其重要性
- 内存覆写导致信息丢失: 固定长度的内存缓冲区在持续压缩过程中会出现信息降级, 难以处理分散在远端的信息
- 监督信号稀疏且滞后: 传统的RL通常仅依靠最终答案的正确性作为单一奖赏, 无法有效知道中间繁琐的内存更新步骤

### 贡献
>[!核心论点] 
> - 历史增强状态机制, 允许Agent生成回调查询以访问历史, 以及多级奖赏设计, 通过结合轨迹级和步骤级信号解决监督稀疏的问题
> - 可以显著提升在超长文本下的推理健壮性

## Method
### 定义
- **任务定义**：给定问题 $Q$ 和正确答案集 $Y$ 。因此我们的数据集就是(Q, Y). 长文档 $C$ 被分块 $c_0, c_1, \dots, c_{T-1}$ 顺序提供给模型

### 历史增强状态机制
- 状态转移的差别![[Pasted image 20260305162531.png]]
	- 可以看到, 左侧多出了一个回调访问, 右侧多出了显式召回记忆的部分
	- 这个检索函数是通过计算查询词语历史内存的单词级别重叠率来选取最相关的历史片段
### 多级奖励设计
![[Pasted image 20260305165244.png]]
- 结果级别奖励
	- 如果答案正确, R=1, 否则为0
- 步骤级别奖励
	- 内存信息增益
		- 测量内存更新前后的信息变化$r_{memory, t}^{(g)} = max_{y \in Y} recall(m_t^{(g)}, y) - max_{y \in Y} recall(m_{t-1}^{(g)}, y)$
		- 鼓励模型在内存中保留更多与答案相关的tokens
	- 回调奖励
		- 评估检索到的内容是否提供了当前上下文之外的额外价值
		- 如果检索出的历史片段叠加当前分块后，相比仅有当前分块显著提升了答案相关词的召回率，则给予奖励
	- 格式奖励
		- 是否输出特定的xml格式
- 优势函数扩展
	- $$\hat{A}_{t}^{(g)} = \alpha \hat{A}_{out}^{(g)} + (1 - \alpha) \hat{A}_{state,t}^{(g)}$$
	- 也就是加入了步骤级别奖励
	- 使用超参数来调节
## Exp
### 数据集基准
- 训练集
	- **HotpotQA**：作为主要训练和同分布测试集，通过填充随机文档将上下文长度扩展至 50 到 6400 个文档
- 测试集
	- **2WikiMultiHopQA**：作为跨域测试集，用于评估模型的泛化能力
- 对比基线
	- **通用 LLM**：Qwen2.5 (3B/7B)、DeepSeek-R1-Distill-Qwen
	- **长文本专用 LLM**：Qwen2.5-1M 
	- **专用记忆智能体**：MemAgent
