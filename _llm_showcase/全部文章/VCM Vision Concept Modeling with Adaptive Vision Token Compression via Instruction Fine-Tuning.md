## Abstract + Introduction
### 背景
- 主流架构: [[LLAVA]]架构已经成为主流, 他们的范式是 ViT+projector+LM
	- 其中ViT, 会用固定的token数来映射图片
- 这种ViT+Projector+LM被证实在各类VQA问题上十分有效
### 挑战
- 计算的静态性: 无论问题的难易, ViT总是会输出固定的token数
- 认知脱节: 人类并不是逐个像素的扫描, 而是关注图片的高级特征
- 可扩展性的挑战: 这种token冗余, 会造成在高分辨率图像或者长视频场景, 造成严重的token冗余, 严重限制了LLM的应用

### 贡献
>[!核心论点] 
> - MLLM不应该通过静态的token来压缩任务, 应该引入"视觉概念模型" 
> - 这种VCM, 能够根据输入的文本指令, 动态的决定要从图像中提取哪些视觉概念
> - 作者证实可以通过一种自监督框架, 无需通过人工标注

## Method
### 动态视觉预算
- 动机: 之前的做法, 要不就是固定token数, 或者是固定压缩比率, 应该采用衡量指令语义的复杂性, 来决定模型应该花费的token
- 进行预实验, 探究指令, 回应与视觉token的相关性
	- **指令 (Instruction)** 中的关键词越多，所需的视觉 token **越少**（因为任务更具体）
	- **回应 (Response)** 中的关键词越多，所需的视觉 token **越多**（因为答案更详细）

- 他们设计了线性公式，根据 $N_{key}$ 来估计目标概念长度 $L$。
$$
L = \left\lfloor M \cdot S \cdot \left(1 - \frac{N_{key} - N_{key}^{max}}{N_{key}^{max} - N_{key}^{min}}\right) \right\rfloor
$$
- 公式解析：
	- $M$ 是原始 token 长度（如 576）。
	- $N_{key}$ 是指令和回问间的关键词数量之差。
	- $N_{key}^{max}$ 和 $N_{key}^{min}$ 是这个差值的归一化范围（例如 10 和 -35）。
	- $S$ 是一个信息域标量 (information domain scalar)，用于控制最大的 token 长度（例如 1/4）。
- 完成了对图像复杂性的建模

### 动态概率的前向-后向传播
- 将这个问题表述为动态规划, 灵感来自语音领域的CTC
- 输入就是之前固定的M个token,  每个token进行二元变量, 决定应该保留还是丢弃
- 目标: 一个由L个保留token扩展得到的扩展序列Z, 为了允许跳过和合并, 我们在中间插入0, 变成$[\emptyset, z_1^V, \emptyset, ..., z_L^V, \emptyset]$, 长度变为2L+1
- 动态规划: 计算从M个token, 映射到这个2L+1的目标序列的所有路径的总概率
- 前向变量$\alpha(t,l)$: 表示前t个token, 映射到前 $l$ 个目标符号序列的概率
- 后向变量$\beta(t,l)$: 表示从 $t$ 时刻开始的剩余输入 token 序列，映射到从 $l$ 开始的剩余目标符号序列的概率
- **总概率 $p(Z^V|Y^V)$**：就是所有可能的、合法的对齐路径的概率之和。这可以通过 $\alpha$ 和 $\beta$ 在任意时刻 $t$ 的乘积求和得到。


- 最终目标就是最大化这个总概率
