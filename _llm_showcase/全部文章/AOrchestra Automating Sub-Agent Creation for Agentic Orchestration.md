## Abstract + Introduction
### 背景
- 多智能体的技术现状:
	- 多代理工作流协作(MAS): 如MetaGPT, 通常依赖固定的工作流或者预定义的角色(产品经理, 架构师), 这些系统通过预设的通信协议让角色协作
	- SubAgent As Tools: 允许主代理想调用工具一样调用子代理

### 挑战
- 预定义的角色具有僵化性, 无法覆盖开放世界中的动态问题
- 上下文爆炸: 如果将所有问题都交给子代理, 会导致信息过载
- 设置固定的角色和工作流, 会需要大量的人工prompt
### 贡献
>[!核心论点] 
> - 子代理应该被视作根据任务需求即时调用的工具
> - 利用中心编排器来分配任务, 完成编排与执行的解耦

## Method
### 动态实例化机制
- 动机: 一个任务的执行者, 只应该拥有完成该任务所需的最小必要信息和能力集
- 工作机制
	- 作者将Agent抽象为四元组$$\Phi = (I, C, T, M)$$
	- **$I$ : 当前子任务的具体目标。
	- **$C$ :** 经过整理的工作记忆。编排器会从历史中筛选出只与当前任务相关的信息，而不是把整个聊天记录丢给子代理 。
	- **$T$ :** 此时此刻需要的工具子集
	- **$M$ :** 适合该任务的模型
- 运作机制
	- 编排器并不直接与环境交互(不执行bash), 他的动作只有两种
		- Delegate: 临时创建一个高度专业化的Subagent来执行任务
		- Finish: 完成任务, 结束返回结果

### 可学习的编排器
- 动机: 创建一个最好的代理就变成了一个优化问题, 作者希望使用数据驱动来让编排器学会使用
- 工作机制
	- SFT, 用户任务编排: 学习分解任务和筛选上下文
	- ICL: 上下文学习, 学习如何选择模型以平衡成本和效果
