## Abstract + Introduction
### 背景
- 主流的大语言模型多智能体系统（MAS）如 **CAMEL**、**AutoGen** 和 **MetaGPT** 已成为多轮推理的实用范式
- 这些系统通过角色专业化、结构化对话和标准化操作程序（SOPs）来提升性能
- 现有的协作模式通常依赖于**固定或密集**的通信模式, 在整个推理过程中使用固定的拓扑结构
### 挑战
- 多轮推理具有阶段性的特点, 早期需要广泛探索, 后期需要针对性验证, 固定拓扑难以适配这种动态需求
- 智能体的协作过程往往是隐形的, 缺乏可解释的手段来追踪通信路径随着推理深度的演变
### 贡献
>[!核心论点] 
> - 作者主张通信拓扑应该视作一个自适应对象, 在每轮推理中动态重构稀疏的DAG
> - 提出DyTopo框架, 利用Agent的Query和Key描述符进行语义匹配, 在每一轮中诱导出最优的通信路径

### 定义
- **智能体集合**：系统由 $N$ 个异构工作智能体 $\mathcal{A}=\{a_{1},...,a_{N}\}$ 组成，每个智能体拥有特定的角色描述 $\rho_{i}$ 和局部内存缓冲区 $\mathcal{H}_{i}^{(t)}$
- 双信道通信
	- **公开信道 ($\mathcal{M}_{pub}$)**：Manager 可见，用于全局分析
	- **私有路由信道 ($\mathcal{M}_{priv}$)**：根据诱导拓扑进行定向传输
- 关键描述符
	- **查询描述符 ($s_{q,i}^{(t)}$)**：总结智能体当前寻求的信息
	- **键描述符 ($s_{k,i}^{(t)}$)**：总结智能体当前能提供的能力
### 每轮智能体执行
- 第一阶段：单次推断与描述符生成
	- 智能体的“输入”：局部状态 $S_{i}^{(t)}$$$S_{i}^{(t)}=[\rho_{i};C_{task}^{(t)};\mathcal{H}_{i}^{(t)}]$$
		- $\rho_{i}$ :角色描述
		- $C_{task}^{(t)}$ :轮次目标
		- **$\mathcal{H}_{i}^{(t)}$**：之前的思考记录和收到的历史信息
	- 智能体的“输出”：输出元组 $O_{i}^{(t)}$$$O_{i}^{(t)}=\langle m_{pub,i}^{(t)},m_{priv,i}^{(t)},s_{q,i}^{(t)},s_{k,i}^{(t)}\rangle \sim \pi_{\theta_{i}}(\cdot|S_{i}^{(t)})$$
		- **$m_{pub,i}^{(t)}$ (公开消息)**：发给 Manager 看的汇报
		- **$m_{priv,i}^{(t)}$ (私有消息)**：包含具体技术细节，只发给目标Agent
		- **$s_{q,i}^{(t)}$ : Query**：用自然语言写出需要什么信息
		- **$s_{k,i}^{(t)}$ : Key**：用自然语言写出提供什么信息
- 第二阶段：同步屏障与上下文更新
	- 系统设置了一个“屏障”，强迫所有智能体停下来 。系统利用所有智能体提供的 $s_{q}$ 和 $s_{k}$ 进行语义匹配，算出这一轮的**动态拓扑图 $G^{(t)}$** 。只有图里连上的边，私有消息 $m_{priv}$ 才能穿过去
	- 路由完成后，智能体才会更新它进入下一轮的记忆 $\mathcal{H}_{i}^{(t+1)}$$$\mathcal{H}_{i}^{(t+1)} \equiv \mathcal{H}_{i}^{(t)} \oplus m_{pub,i}^{(t)} \oplus \Sigma_{\sigma_{i}^{(t)}}(\{m_{priv,j}^{(t)}|j\in\mathcal{N}_{in}^{(t)}(i)\})$$
