## Abstract + Introduction
### 背景
- 手工设计为主: 以Prompt工程构建的Agent架构
- Prompt优化: 让LLM自动优化Prompt, 但这仅限于优化指令, 无法改变权重
- 全代码搜索: 如ADAS, 尝试无限的代码空间中搜索整个Agent系统

### 挑战
- 设计不可复用: 手工设计的Prompt Agent针对特定任务, 但不具备很好的泛化性
- 搜索空间的复杂度过高: 如果只搜索Prompt, 上限过低, 如果搜索任意代码, 搜索空间过大, 且评估成本过高
- 缺乏同一的接口协议,导致优秀模块无法集成
### 贡献
>[!核心论点] 
> - 通过定义一个标准化的模块化设计空间, 让Agent抽象为几个核心功能模块, 利用LLM进行进化搜索和重组, 从而自动发现超越人类专家设计的sota Agent
> 	- 将Agent解耦规划, 推理, 工具使用, 记忆四大模块, 并标准化其输入输出
> 	- 混合搜索策略: 使用模块进化(更新模块)和模块重组(使用原有模块)
> 	- 性能预测器: 使用代理模型来预测Agent性能, 降低搜索成本

### 模块化设计空间
- 动机: 他们发现现有的Agent(如AutoGPT, Voyager), 都可以被拆分为几个核心通用组件, 尝试标准化接口
- 工作机制
	- Agent 被定义为一个元组 $A=(P, R, T, M)$。系统预定义了四大模块的标准 I/O 接口
		- **Planning ($P$):** 输入任务 $d$ 和反馈 $f$，输出子任务序列 $s_1, ..., s_n$。负责将复杂问题分解
		- **Reasoning ($R$):** 输入子任务 $s_i$，输出推理结果 $r_i$
		- **Tool Use ($T$):** 输入问题，从工具池 $\tau$ 中选择工具并执行。负责扩展能力边界
		- **Memory ($M$):** 负责 $M_{write}$ (写入) 和 $M_{retrieve}$ (检索)。负责长期记忆和经验积累
### 模块进化
- 动机: 单纯的Prompt优化容易陷入局部最优, 随机组合又低效, 作者引入AgentSquare, 旨在同时利用探索和利用结合优秀模块
