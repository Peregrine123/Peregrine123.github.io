## Abstract + Introduction
### 背景
- RLVR是提升LLM推理能力的主流Post-training范式
- 典型代表如R1使用的GRPO算法, 通过生成多个rollout并对正确答案给予奖励, 促使模型生成复杂的CoT
- Self-training方法(如STaR), 通过模型生成的正确理由上进行微调来增强推理
### 挑战
- 结果导向的奖励容易造成奖励稀疏的问题
- 奖励作弊: 模型可能会学习到错误的推理模式, 但是只要能够凑出正确的答案即可

### 贡献
>[!核心论点] 
> - 尝试不借助显式的RL算法, 仅通过self-distillation来显著提升LLM的LongCoT能力
> - 利用同一模型在不同语义环境下(有提示词 vs. 无提示词) 扮演教师和学生脚色, 通过教师模型对自身生成的正负样本进行反思, 纠错并生成高质量的逻辑分布, 再将其蒸馏给学生
> - 提供了一种计算高效的方案

## Method
### 候选答案的采样与生成
- 系统从原始监督数据集 $\mathcal{D}=\{(q_{i},a_{i})\}_{i=1}^{N}$ 出发，仅利用问题 $q_i$ 和最终答案 $a_i$ 进行自动标注
- 多路采样
	- 使用预设的“专家导师”系统提示词 $sys_{inf}$ 指导基座模型 $f_{\theta}$
	- 在采样温度 $T_{roll}$ 下执行 $\overline{K}$ 次随机 rollout ：$$\{r_{k}\}_{k=1}^{K}\sim f_{\theta}(sys_{inf},q_{i};T_{roll})$$
- 样本分区
	- 根据最终答案, 将生成的相应分为两类
		- 正确集: 包含正确答案的响应轨迹
		- 错误集
- 代表性样本选取
	- 从正确集中随机选取一个轨迹$r^{corr}$
	- 从错误集中选取出现频率最高的错误答案对应的轨迹$r^{wrong}$, 作为模型典型认知错误的代表

### 教师端的语义精炼
- 通过构建语义对比语境强制模型自我修正
- 构建语义提示
	- 想模型提供一个包含对比视角的用户提示$u_{SSB}$
	- 该提示同时展示 $r^{corr}$（正确路径）和 $r^{wrong}$（典型错误路径），并告知模型其中一个正确，一个错误
- 生成鲁棒解 ($\tilde{r}_{i}$)
	- 要求模型作为教师，生成一份**独立且鲁棒**的详细推导
	- **关键约束**：教师需在推导中针对潜在错误逻辑进行警戒，但严禁在文本中提及自己参考了学生的轨迹
	- 最终生成的 $\tilde{r}_{i}$ 必须经过验证，确保其最终答案与 $a_i$ 一致

### Logits 预计算与存储
- 为避免训练时的重复计算, 作者采用离线存储策略
- 软标签提取
	- 将教师端的完整对话--系统提示 + 语义提示 + 最终生成的鲁棒解--输入模型
	- 仅提取**回答部分**每个 token 对应的原始 Logits 序列 $l_i = [l_i^1, l_i^2, \dots, l_i^{|\tilde{r}_i|}]$ 并存入磁盘

### Logit 级别蒸馏训练
- 在训练阶段，模型不再作为教师，而是作为学生在“无提示”环境下学习
- 学生端视角
	- 学生模型仅接收 $sys_{inf}$ 和原始问题 $q_i$
	- 强制模型生成与教师生成的 $\tilde{r}_{i}$ 相同的 token 序列，并计算学生端在这些位置上的 Logits 序列 $\hat{l}_i$. 
	- 学生模型是按照老师给的标准答案进行逐个logits生成, 并非完全生成整句话
- 知识蒸馏损失函数 (KD Loss)
	- 训练目标是最小化学生与教师输出分布之间的 **KL 散度** ：$$\mathcal{L}=\frac{1}{|\mathcal{T}|}\sum_{i}T_{kD}^{2}\frac{1}{|\overline{r}_{i}|}\sum_{j}KL(softmax(\frac{l_{i}^{j}}{T_{kD}})||softmax(\frac{\hat{l}_{i}^{j}}{T_{kD}}))$$
	- **纯软标签监督**：不使用任何硬标签（Hard Labels）的交叉熵损失，仅通过微调 Logits 分布来保留模型的泛化能力
