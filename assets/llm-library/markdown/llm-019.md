## Abstract + Introduction
### 背景
- 存在预训练知识依赖: 主流LLM擅长利用预训练记忆来处理Prompt中的推理任务
- In-Context-Learning和RAG的局限性: 主流的ICL利用few-shot让模型学习任务格式, 而RAG侧重于外部上下文的检索与组织
- 现有的Context Engineering 默认只需要把信息召回, 模型就能够自动利用, 作者认为这忽略了最基础的底层能力: 模型能否学会召回的信息
### 挑战
- 动态学习能力的缺失: 真实的复杂任务高度依赖上下文, 模型需要从任务特定的背景中学习预训练阶段从未见过的新知识, 并进行演绎推理, 而不仅仅是做长文本的信息检索或阅读理解
- 评测维度的缺失: 现有的长文本或指令遵循基准测试, 往往将检索能力和真正的知识内化能力混为一谈, 导致无法准确定位模型的技术瓶颈

### 贡献
>[!核心论点] 
> - 当前大模型在真实场景下的上下文学习能力严重匮乏, 这是限制其解决复杂任务的核心瓶颈
> - 将大模型评测的焦点从静态记忆提取转向动态新知识内化

### 上下文分类体系
![[Pasted image 20260304094219.png]]
- 作者将上下文划分为四大体系
	- 领域知识推理
		- 要求模型依据专业知识进行专业咨询或案件裁决
	- 规则系统应用
		- 引入全新的形式化规则, 考察模型的演绎推理能力
	- 程序性任务执行
		- 提供复杂的工作流或操作指南, 要求模型执行故障排除或运营指导
	- 经验发现与模拟(最难)
		- 从海量实验数据或观测记录中归纳出底层规律, 或者在虚拟盲盒中进行推理

### 基准测试构建
- 专家驱动流程
	- 阶段一: 专家设计包含新知识的上下文, 确保信息的真实感和充分性
	- 阶段二: 为每个上下文设置对应的任务, 确保这些知识需要从提供的上下文进行学习, 并且尽可能按照顺序执行逻辑
	- 阶段三: 专家撰写详细的任务评估指南, 详细的二元验证准则
- 无污染设计
	- 虚构创作: 完全原创的法律体系或者编程语言
	- 修改既有事实: 篡改历史时间或者改变物理常数
	- 纳入小众与新兴知识: 纳入最新论文, 或者来自狭窄知识领域等未纳入预训练集的数据

### 基于规则的自动化评测
- 二元准则设计: 每项任务平均关联16.6条验证准则, 每条准则仅允许是/否判定
- 严苛的评估标准: 使用全有或全无机制, 模型必须完全通过所有关联准则才能获得得分
- 使用GPT-5.1作为评测器

## 结果分析
- 不同任务上模型的表现差异
	- 最易类别: 领域知识推理是相对容易处理的, 最优模型得分为25.3%
	- 最难类别: 经验发现与模拟评分最低, 平均解决率11%
	- 具体子类解析
		- 规则化系统应用中, 法律监管类表现比较好, 但是数学形式化极难, 大多数模型解决率低于15%
		- **经验发现与模拟**中，**模拟环境**子类是重灾区，绝大多数模型得分低于 **11%**

- 错误类型![[Pasted image 20260304102742.png]]

- 归纳推理与演绎推理具有显著差别
	- 前三类任务侧重演绎应用, 而第四类任务侧重于归纳推断(也就是从数据中发现规律)
	- 模型在归纳任务上的表现比演绎任务低6%

- 异常情况分析
	- 版本退化发现: GPT-5.2相较于GPT-5.1反而出现了退化, 分析显示5.2在处理长文本时因果链一致性较差, 而且更容易违反显性约束
- 能力发现: 
	- 强大的长文本处理和指令遵循能力是上下文学习能力的必要不充分条件, 即使具备以上两种能力, 依然可能在CL-bench上失效

## 拓展分析
### 失败模式
- 上下文误用是最普遍的失败模式, 误用率超过30%
- 上下文忽略: 该错误率与模型整体性能高度相关
- 格式错误: 格式错误的占比也非常高, 反映出模型在复杂上下文干扰下指令遵循能力的衰减

### 推理算力的边际递减
- 增加推理计算量对上下文学习能力提升并不显著

### 文本长度与知识类型的双重挑战
- 随着上下文长度增加, 所有模型性能一致性出现下滑, 当Claude Opus 4.5在文本从0-15k拓展至 120k, 解决率跌出20%
- 知识结构差异性: 即使在同一领域, 不同的细分知识类型也表现出巨大的差异

## 讨论
### 上下文学习的意义
- In-Context Learning vs. Context Learning
	- 第一种是通过少量示例让模型理解任务格式
	- 第二种则是从上下文中获取新知识, 并用已有推理能力解决新任务
	- 有点像人的思维模式, 通过少量新知识获得新的领域能力
- 他的作用
	- 上下文是链接静态参数知识和动态现实任务的桥梁
	- 它代表即使学习的能力, 可以不更新模型参数下, 快速适应新领域

### 提升上下文学习的路径
- 使用上下文感知的数据
	- 构建包含违建知识的训练样本, 迫使模型必须依赖上下文而非预训练知识
	- 通过领域文档+任务配对的方式生成训练数据, 强化模型对上下文的依赖

- 课程学习策略
	- 从简单子任务开始, 逐步增加上下文复杂度
	- 用于提升模型的长上下文处理能力和指令遵循能力

- 自动生成细粒度评估标准
	- CL-bench依赖人类专家提出评估标准
	- 未来可以开发自动生成高质量rubrics的方法, 用于训练中的奖励信号
