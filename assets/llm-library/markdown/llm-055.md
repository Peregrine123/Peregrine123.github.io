## Abstract + Introduction
- 背景
	- NLP领域指令微调的重大进展: 通过高质量的, 机器生成的指令-遵循数据对, 能够对预训练LLM进行微调, 极大的提升模型zero-shot能力以及人类意图遵循能力
	- 多模态领域: 我们已经有[[CLIP]]这样强大的预训练视觉编码器, 以及[[Flamingo]]和[[BLIP]]这样的多模态模型
- 挑战
	- 当时的多模态大模型没有很好的通用性, 导致不能像GPT一样和他们自然的聊天, 而是应该有任务导向的prompt
	- 多模态数据的匮乏: NLP之所以能够进行大规模指令调优, 是应为高质量的大模型可以近乎免费额生成, 而对于多模态来说, 需要图像, 指令, 文本的三元组, 既没有模型能够直接生成, 人工标注耗时又昂贵
- 贡献
	- 核心观点: 决定多模态大模型的进展, 不是复杂的模型架构, 而是优质的大规模视觉指令微调数据
## Method
### 构造视觉微调数据
- 动机: 我们诱导纯文本的GPT-4模型, 通过[[In-context learning]], 生成对图像的描述
- 我们先从COCO图像中提取图像描述与边界框
- 构建prompt: 我们先人工进行标注, 构成[[In-context learning]]的高质量样本
- 指令生成: 我们利用few-shot示例, 生成三种类型的样本对
	- 多轮对话
		- 模拟用户对图像内容的多轮回答
	- 详细描述
		- 对图像的各个物体, 位置, 关系进行详细描述
	- 复杂推理
		- 根据图片内容提出复杂逻辑问题

### 两阶段微调
- 我们利用[[CLIP]]训练的ViT模型作为视觉编码器, 以及优秀的多模态模型Vicuna作为语言编码器
- 我们引入一个简单的线性层
#### 第一阶段
- 我们将 ViT和LM模型都进行冻结, 只更新线性层的参数
- 线性层用于对齐ViT模型和LM模型的输出
#### 端对端微调
- 我们依然冻结ViT模型
- 我们对LM和线性层都放开权重更新
- 我们使用刚才获得的视觉微调数据集进行更新, 得到最终的模型
