## Abstract + Introduction
### 背景
- 工业界出现了Devin那样的优秀作品, 但由于是闭源的, 学界无法复刻
- 框架碎片化: 市场上的许多Agent框架(AutoGPT, MetaGPT, LangChain)
- 工具调用的局限性: 大多数框架依赖JSON风格的调用或者简单的API封装, 限制了Agent处理开放世界, 复杂问题的能力
### 挑战
- 缺乏统一的运行时环境
- 缺乏在同一环境下, 标准化的BenchMark
- 复现性危机: 由于在不同的环境下, 容易出现复现不易的现象
### 贡献
>[!核心论点] 
> - 必须定义一个和人类一样的开发环境, 能够通过编写代码, 使用命令行和浏览网页来与世界交互
> - 构建了一个基于事件流的开源平台
> - 提供了一个安全隔离的Docker环境, 允许Agent任意修改代码
> - 集成了15个主流基准测试, 实现了一站式的评估
## Method
### 基于docker的安全运行环境
- 动机
	- 如果Agent对环境进行不可逆的操作, 容易造成环境破坏
	- 我们需要构造一个基于状态的, 让Agent能够像人一样在环境中连续交互
- 事件流
	- 系统核心的数据结构是事件流, 他是一个严格按照时间排序的列表, 记录了所有的action和observation
- Docker沙箱, 每一个人物都会独立启用一个容器
- 运行时客户端: 运行在Docker内部的REST API服务器, 他负责接收Agent的指令, 在容器内执行, 并且将结果封装为Observation反馈

### CodeAct代理范式和AgentSkills库
- Agent仅使用极少数的核心动作类, 而非结构化的Json
- 通用动作函数
	- `IPythonRunCellAction`：执行任意 Python 代码 。
	- `CmdRunAction`：执行任意 Bash 命令 。
	- `BrowseInteractiveAction`：执行浏览器操作代码 。
- 即时动作创建
	- 当需要解决特定子任务, Agent直接编写对应python函数并执行, 这使得Agent具有动态工具创建的能力
- Agentskills技能库
	- 为了补充纯代码难以实现的复杂功能, 作者封装了一个AgentSkills的Python库, Agent可以直接import
