## Abstract + Introduction
- 背景
	- Post-Training成为业界继续提升能力边界的共识, [[SFT]]和[[RLHF]]是主流范式
	- [[CoT]]是主流范式, 是提升推理性能的关键手段
	- 现有的探索方向
		- 基于过程奖励模型(PRM)
		- MCTS
		- 基于SFT的强化学习
- 挑战
	- 对高质量[[SFT]]数据的严重依赖
	- 自主演进的能力缺失: 模型真的需要人一步步教才能学会思考吗? 就不能让模型自己探索吗
- 贡献
	- 核心观点
		- 复杂的高阶推理模型, 可以通过大规模的, 纯粹的RL来激发模型的潜能, 让模型涌现推理能力
	- Deepseek-R1-zero: 通过无[[SFT]], **纯粹的大规模RL**, 展现出了强大的自我推理和反思能力
	- 高效蒸馏
	- 少量SFT+大量RL, 就能让模型涌现推理能力
## Method
### 动机
- base模型可以在不经过[[SFT]]下, 直接进行RL获得推理能力
- 我们能否绕过SFT, 直接使用RL呢? 因为构建SFT是昂贵且复杂的
### 奖励模型
- [[lecure 16 RL#Deepseek-R1]]
- 借助[[GRPO]]

### 运行机制
#### SFT冷启动
- 使用数千条高质量的长CoT数据对base模型进行[[SFT]]
- 目的: 让模型知道推理模式是如何进行的
#### 面相推理的RL
- 使用[[GRPO]]进行继续优化
- 相较于Deepseek-R1-zero, 新增了思考时语言一致性奖励, 提升可读性
#### 拒绝采样与二次SFT
- 当第二阶段的RL模型变得非常强大后, 用他来生成海量候选答案
- 通过**拒绝采样**, 保留推理正确且格式良好的答案, 构建出一个规模巨大(60w)且高质量的[[SFT]]数据集
- 利用这个高质量数据集, 重新对原始的基础模型进行[[SFT]], 相当于巩固知识
#### 全场景RL
- 对第三阶段的模型进行RL, 目标是泛化
- 此时的奖励信号是混合的: 对推理任务使用**规则奖励**，对通用对话、写作等任务，则使用传统的、基于**人类偏好训练的神经奖励模型**


### 高质量蒸馏推理数据集对小模型具有提升作用
- 我们使用从第一阶段RL产生的60w推理SFT和20w非推理数据集, 构成一个高质量数据集
- 然后我们对小模型进行微调, 尺寸从1.5B到72B
- 发现模型的推理能力都得到了显著的提升, 尤其是在code和math方面提升巨大
