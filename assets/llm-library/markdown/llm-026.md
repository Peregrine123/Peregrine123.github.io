## Abstract + Introduction
### 背景
- VLM模型在提升性能的途中, visual encoder成为性能瓶颈

### 挑战
- 技术流派
	- 基于平铺的方法-InternVL2.0
		- 将高清图片切分为多个块
		- 会导致超长的上下文
	- 基于自分辨率的方法-[[Qwen-2.5 VL]]
	- 双塔架构: 显式构建高分辨率和低分辨率的模型

### 贡献
>[!核心论点] 
视觉模态不仅仅应被作为LLM的眼睛, 还应该处理上下文压缩
>证明了将1D的文本序列映射到2D的图像序列并进行压缩是可行的, 这为解决LLM的长上下文提供方向


## Method
### DeepEncoder
- 动机
	- 能否使用局部注意力处理高分辨率的局部信息, 使用全局注意力处理低分辨率信息
- 阶段一: 感知
	- 以1k x 1k 图像为例 先切分为16x16的patch, 得到4k个patch
	- 这 4k 个token被送入到编码器中, 使用局部注意力进行处理
	- 所以这个阶段激活显存是可控的
- 阶段二: 压缩
	- 将SAM输出的4k token输入到16x的下采样卷积压缩模块
	- token数量减少到256个
- 阶段三: 知识
	- 将这256个高度压缩token输入到CLIP-large编码器
	- 应为只有256, 所以可以轻松的进行全局注意力
	- 最终, 仅使用256个视觉token交给LLM

## 启示
- Deepseek-OCR 提出了一种LLM的上下文工程的思路
	- 将1D的文本升维到2D的图像token
	- 也许在大海捞针的任务里面表现不好
	- 但如果我们的目的是获得统摄性的理解, 那就已经足够了
	- 我们可以在进行k轮对话后, 将LLM的历史渲染为一页纸
	- 我们可以对历史分段, 并进行渐进式的模糊记忆
