## Abstract + Introduction
>[!核心论点] 
> - 旨在填补端到端仓库生成的空白, 并且该Benchmark聚焦于微服务级别的任务生成
> - 构建包含 106 个仓库（涵盖 18 个领域、11 个框架）的多語言基准，并采用 **Pass@1**、**API 覆盖率 (AC)** 和 **部署成功率 (DSR)** 三维评估体系


## Exp

- Pass@1

#### DSR & AC
![[Pasted image 20260215115010.png]]
- **有表无里”的开源 Agent ：** MetaGPT (Claude) 的 API 覆盖率高达 **73.04%** (Python) 和 **74.14%** (Java)，几乎与商业 IDE 持平
- **部署瓶颈：** 尽管 API 骨架完整，MetaGPT 的部署成功率 (DSR) 仅为 **13.00%** (Python)，这意味着其生成的代码往往因基础设施配置错误而无法运行
- **商业 IDE 的工程优势：** Copilot 在 Python 上的 DSR 达到了惊人的 **95.45%**，展现了极强的环境配置与启动脚本生成能力

### 失败模式分析
- **跨文件一致性 (50.2%) ：** 模型最常在不同文件间的 API 定义、数据模型和错误处理逻辑不统一上出错
- **架构连贯性 (26.0%) ：** 无法将高层需求转化为合理的模块分解和代码组织结构
- **依赖管理 (23.8%) ：** 频繁出现环境配置错误、库版本冲突或遗漏必要依赖文件
