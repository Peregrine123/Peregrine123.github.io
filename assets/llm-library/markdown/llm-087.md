## Abstract + Introduction
### 背景
- 知识蒸馏是一种提升小模型的普遍方法
- 业界普遍认为, 提供更长, 更复杂, 逻辑更严密的推理路径(如长CoT)能持续提升模型能力
### 挑战
- 小模型无法真正消化强力模型输出的复杂CoT
- 能力瓶颈: 当教师模型与学生模型存在巨大的参数量差异时, 传统的蒸馏方法可能失效
### 贡献
>[!核心论点] 
> - 作者发现, 小模型反而更倾向于学习与自身计算容量相匹配的, 较短且简单的推理链
> - 引入混合蒸馏策略, 通过将长短CoT示例以及强弱教师的输出按比例混合, 平衡推理复杂性
> - 解释了知识蒸馏中复杂性匹配的重要性

## Method
### Mix-Long 
- 定义: 将长思维链和短思维链进行混合
- 目标: 让小模型在接触到详细推理步骤的同时, 不至于因序列过于冗长而陷入逻辑混乱
### Mix-Large
- 定义: 将强模型教师和弱模型教师的响应混合
- 目标: 利用弱教师提供认知阶梯, 帮助小模型内化大教师的高阶推理能力

### 构造细节
- 数据构造
	- 教师模型通过拒绝采样和贪婪编码生成推理序列
	- 筛选出不同教师均能回答正确的题目-答案对, 以消除题目难度带来的干扰
- SFT
	- 使用LLaMA-Factory框架进行有监督微调
	- 参数量<14B, 全参数微调
	- 参数量>= 14B, 采用LoRA微调
- 关键超参数 $\alpha$
	- 作者进行了消融实验发现，**$\alpha = 0.2$** 是小模型的最优平衡点
	- 这意味着 20% 的高难度数据配合 80% 的简单数据能产生最佳的蒸馏效果

- 启示
	- 小模型并非不能学习强力推理, 而是不能接受剧烈的分布偏移, 文章通过构建短CoT和弱教师模型作为桥梁, 让模型循序渐进的学习
