## Abstract + Introduction
- 背景
	- 当时微调存在着几种技术路线
		- 全参数微调: 以GPT为典例, 缺点是需要加载所有的模型参数, 对GPU显存要求极高(LLAMA 65B 需要750B的显存)
		- 参数高效调优: PEFT系列, 以[[LoRA]]为例, 仅在注意力层增加可训练的低秩矩阵, 极大减少模型显存, 但原始的16-bit模型本身依然占据巨大空间(65B需要花费130GB显存)
		- 模型量化: 主要用在模型推理将模型从16bit转为8bit甚至4bit, 显著降低模型的推理成本
- 挑战
	- 模型量化的方案大多为推理设计, 在训练场景下, 一涉及到反向传播容易break down
	- 性能的不可能三角
		- 全参数微调必须花费大显存
		- PEFT依然受制于基础模型的巨大显存
		- 量化牺牲了模型的可微调性
- 贡献
	- 核心论点: 我们可以在被深度量化到4-bit的模型上, 对冻结的权重进行微调, 并实现接近全参数微调的结果
	- 4-bit NormalFloat (NF4): 正太分布的数据类型, 专为神经网络设计
	- 双重量化: 
	- 分页优化器
## Method
- [[模型量化]]
### NF-4
- 动机: 在神经网络中, 数据并不是均匀分布的, 而是呈现正太分布, 使用均匀的尺子衡量会造成进度浪费

$$
q_i = \frac{1}{2} \left( Q_X\left(\frac{i}{2^k + 1}\right) + Q_X\left(\frac{i+1}{2^k + 1}\right) \right)
$$

我们来逐一分解这个公式的每个部分：
- **$k$**：代表我们想要的比特数。对于 NF4，$k = 4$。
- **$q_i$**：代表我们要计算的第 $i$个量化值。$i$ 的范围是从 0 到 $2^k - 1$。对于 4-bit，我们总共需要计算 16 个值（$q_0, q_1, ..., q_{15}$）。
- **$Q_X(p)$**：这是最核心的部分，它代表标准正态分布的分位数函数 (Quantile Function)。
    - 简单理解：你可以把它看作是累积分布函数 (CDF) 的反函数。
    - $CDF$ 回答的是：“给定一个值 $x$，小于等于 $x$ 的数据占比（概率）是多少？”


### 双重量化
- 第一次对权重进行量化, 量化过程会产生32bit的量化常量
- 第二次我们把所有量化常量收集起来, 进行第二次量化, 他们同样会产生32bit的量化常量
- 最终, 我们使用计算来换取了空间
