## Abstract + Introduction
- 背景
	- 存在两种主流的VLM架构
		- 以[[CLIP]]为代表的, 双编码器结构, 为文本和图像单独分配编码器, 在理解型任务上表现优秀
		- 以SimVLM为代表的融合编码器方案, 将图像特征融入到生成式语言模型架构中, 在生成式任务上表现良好
- 挑战
	- 图像的理解和生成两类任务存在矛盾, 如何设计一种架构来兼容这两种任务
	- 数据质量高于数据数量
- 贡献
	- 核心论点: 提出一种统一理解和生成任务的框架, 并且可以通过设计的数据筛选方法保证数据的质量
	- 模型层面: 提出多任务统一架构, 同时做到单模态编码器, 图文编码器和图文解码器
	- 数据层面: 涉及了数据集bootstrap流程, 利用模型生成高质量的合成标题, 提高预训练数据的质量
## Method
### 一套参数, 三次复用
- 共享
	- 图像ViT
	- 文本Transformer共享嵌入层, FFN, 跨注意力CA
- 不共享
	- 自注意力SA
![[Pasted image 20251018110126.png]]
### 图像文本对齐 ITC
- 图像模块: 使用ViT产生图像特征向量
- 文本模块: 使用去掉因果编码的Transformer, 编码文字特征向量
- 将两个向量放到ITC中进行对齐

### 图像引导的文本编码器
- 我们将ViT得到的图像特征向量, 输入到跨注意力CA中
- 跨注意力将文本特征与图像特征融合计算
- 最终输出一个二元标量, 判断文本与图像是否匹配
- 引导模型学习文本与图像的细粒度对齐

### 多模态语言建模
- 这次我们使用标准的Transformer的Decoder块, 并且使用图像ViT的图像特征向量作为跨注意力的输入
- 目标是根据图像自回归的生成文本
- 他的结构几乎和编码器一样, 只是加入了因果掩码

### 数据bootstrap
- 使用预训练好的MED模型, 在COCO上进行SFT, 得到图像描述器以及图文判断器
- 生成: 对每一张网络图片, 使用预训练模型生成高质量的标题
- 过滤: 使用图文判断器进行打分, 判断生成标题是否值得替换旧标题
- 重组数据集
- 重新训练
