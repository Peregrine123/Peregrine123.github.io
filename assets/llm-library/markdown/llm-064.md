## Abstract + Introduction
### 背景
- 函数级编程
- 修复与回归测试
- 代码补全
- 带脚手架的生成: 即使是Commit0这样从头构建的任务, 也往往预先提供了项目结构和函数签名作为强先验信息
### 挑战
- 长程依赖和一致性: 现有基准往往缩减了任务的规模, 使得Agent不需要进行持续数百轮的推理, 规划和执行
- 从零构建的复杂性: 需要涉及架构设计, 依赖管理, 多模块逻辑实现以及打包, 不再是代码块
- 评估的主观性: 缺乏像pytest这样严格, 基于执行的功能正确性验证

### 贡献
>[!核心论点] 
> - 长程软件库生成（Long-Horizon Repository Generation）是通往自主软件工程的关键瓶颈，而当前的 Coding Agents 远未掌握这一能力
> - 在仅凭一份自然语言需求文档、从零构建可安装 Python 库的挑战下，平均测试通过率也**低于 40%**

## Method
#### 领域覆盖 
任务涵盖了 9 个不同的 Python 应用领域，包括 ：
- **系统工具 (System Tools)：** 24 个任务。
- **数据分析与处理 (Data Analysis & Processing)：** 18 个任务。
- **Web 开发 (Web Development)：** 10 个任务。
- 以及网络工具、数据库交互、机器学习等。
#### 难度分级 (Difficulty Levels)

作者根据代码行数 ($LOC$) 将任务划分为三个等级 ：

| 难度等级 | $LOC$ 范围 | 任务数量 | 
| :--- | :--- | :--- | 
| **Easy** | $< 1500$ | 26 | 
| **Medium** | $1500 - 4000$ | 46 | 
| **Hard** | $\ge 4000$ | 32 |
