## Abstract + Introduction
- 背景
	- [[GPT-1]]开创了预训练微调范式, 并广泛取得了验证
- 挑战
	- 泛化能力较弱, 在SFT训练的范围内, 模型表现良好, 但是当[[out-of-distribution]]时, 性能就会急剧下降
	- 多任务学习的瓶颈: 为多个任务设置不同的损失函数依然十分繁琐, 我们需要寻找一种通用的方式来解决
	- 对监督数据的重度依赖: 监督数据费时昂贵, 需要找到一个可扩展的方法
- 贡献
	- 核心论点: 一个[[大模型参数量]]足够大的模型, 能够在经过高质量数据的训练后, 在不经过监督训练下, 对多种通用任务都有很好的效果
	- 这依赖于
		- 模型规模的大幅度上升: gpt-2是1.5B的规模
		- 高质量, 大规模的复杂数据集构建
## Method
### 将NLP任务进行统一语言建模
- 动机
	- 我们为适配不同任务, 而使用不同的任务头并展开不同的训练不够优雅, 我们能否找到统一的框架来描述所有的任务呢
	- 所有的任务都能够根据上下文预测下一个词的问题, 这就是语言模型的定义
- 我们不通过显示的建模多任务学习目标, 而是通过构建多任务的数据集, 让模型通过最大似然, 自学习通用的表征
- 执行任务的形式变成了**条件执行**: 给出prompt, 模型分析prompt执行对应任务

### WebText数据集
- 动机: 数据质量才是关键
	- 使用common-crawl虽然数据量大, 但是信噪比低, 容易让模型学到有毒信息
	- 我们需要建立一个高质量, 领域广泛且经过筛选的数据集, 构建这个数据集的目的不是纯粹的文本, 而是能够让模型学习到隐式的任务
- 构造
	- 最终得到了40GB, 约为800万个文档的容量的数据集
	- 使用了[[BPE]]来构建
## Exp
### 实验设置
- 我们对模型进行WebText训练 得到了1.5B的GPT-2, 并在显著超出WebText数据集上的其他数据集上进行验证

- 在需要世界知识的任务上, 取得较高的成绩

## 启示
- 模型能从大规模高质量数据集上涌现能力的现象得到验证
- 证明了数据质量才是关键的观点
- 无需为NLP建立各种复杂的任务, 只需要一个统一的, 高质量的数据集
