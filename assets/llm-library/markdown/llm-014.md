## Abstract + Introduction
- 背景
	- 当时存在两种技术范式
		- ELMo为代表的BiLSTM架构的语言模型
		- 微调式的GPT以[[Notion/领域(Area)/论文阅读/经典深度学习/脑电Transformer]]架构为核心的语言模型
- 挑战
	- 单向性局限(但是时代还是选择了GPT架构)
		- 无论是ELMo还是GPT, 都是单向语言模型
		- NLP任务仅仅依靠单侧的上下文是次优的
		- 双向是很难的: 预测语言模型时, 如果他能够看到左右的词, 容易造成信息泄露
- 贡献
	- 一个通过**深度双向**预训练得到的语言表征模型，其能力远超单向模型或单向模型的浅层组合
	- 设置了基于掩码的预训练任务:
	- 基于下一个句子的预测
	- 采用预训练-微调的范式

### 掩码语言模型
- 动机: 让模型学会填空
- 实现机制
	- 对于输入的句子, 随机选择15%的单词进行处理
	- 在这些被处理的单词中 遵循80-10-10的原则
		- 80% 替换为特殊token: `<mask>`
		- 10% 替换为随机token
		- 10% 维持不变
- 目的: 基于上下文, 能够准确的得到被处理的词

### 下一句预测
- 动机: 在NLP领域, 许多任务, 如QA和自然语言推断NLI都是判断句子对的关系, 所以应该显式的建模句子之间的关系
- 数据构建(判断是否为下一句)
	- 正负样本各50%
	- `[CLS] Sentence A [SEP] Sentence B [SEP]`
- 是一个简单的二分类任务
