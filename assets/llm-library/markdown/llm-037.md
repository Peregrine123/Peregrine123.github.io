## Abstract + Introduction
- 背景
	- [[GPT-3]]为代表的大模型展现了强大的[[In-context learning]]能力
	- [[CLIP]]为代表的对比学习模型是主流, 他们通过从海量数据中得到强大的Zero-shot和Retrival能力
- 挑战
	- [[CLIP]]能力强大, 但只能进行判别式, 而无法进行生成式任务, 即无法让[[CLIP]]做出图片的文本描述
	- 如何将LLM的强大文本生成能力, 嫁接到图像模型上呢
	- 文本是一维序列, 而图像却是高维的, 而且图像中的文字又是交错混杂的, 如何进行对齐是个问题
- 贡献
	- 核心论文: 将LLM冻结起来, 然后使用图像解码器生成视觉token, 融入到LLM的上下文, 做到在不修改LLM权重下, 让LLM获得图像能力, 利用[[In-context learning]]

### 门控交叉注意力机制
![[Pasted image 20251021095110.png]]
- 在每个冻结的LM层中, 插入门控交叉注意力层
- 我们将视觉输入和语言输入作为KV和V输入
- 通过门控计算, 输入给LM层

### 视觉模块
- 动机: 传统的视觉编码器的结果, 可能是32 x 32的尺寸, 太过庞大不便于输入交叉注意力
- 我们先经过一个预训练的ViT
- 然后我们通过视觉重采样模块, 压缩到64个tokens
- 无论大小, 我们最终都采样为64个tokens

### 多模态交错的上下文
- 输入的序列: `<图像1> "这是一只狗" <图像2> "这是一只猫"`
- 我们的处理: 只交出紧邻的图像, 而把过去的历史图像屏蔽掉, 让LLM专注于现在的图像
- 离历史上下文被抛弃了吗?
	- 并不会, 他隐式的注入在历史文本中了

### 数据训练

- 模型学习模型交错文本的能力

#### 图文对
- 专注于精确图文的匹配

#### 上下文few-shot prompt激活
- 提问模版: `[<图像1>, "问题: 这只猫在干嘛? 回答: 睡觉。"]` `[<图像2>, "问题: 图中有几只动物? 回答: 3只。"]` `[<你的查询图像>, "问题: 图中有几只动物? 回答:"]`
