## Abstract + Introduction
- 背景
	- CV图像识别领域往往采用的是Imagenet预训练-微调的范式, 所有的SoTA都是遵循这个路径
	- 图像识别往往是使用预先定义好的分类器
	- 业界共识是: 模型性能往往依赖高质量的人工标注数据
- 挑战
	- 通用性与泛化性差
		- 只要不存在数据集中, 就需要自己去再次采集, 昂贵费时
	- 监督信号的瓶颈: 高质量人工标注数据集虽然精确, 但是覆盖的范围终究有限, 互联网存在海量的信息, 传统方法无法有效利用这些信息
	- Zero-shot能力较差, 比较依赖于类别标注
- 贡献
	- 核心论点: 不在需要依赖人工标注固定类别的数据集. 通过简单的对比学习目标, 让模型直接从互联网海量数据集中提取的图像-文本对中学习, 就足以训练出一个强大的, 具备卓越零样本迁移能力的通用视觉模型
	- 构建了一个超大规模的图文数据集(400M对)
	- 高效对比学习方法(CLIP): 其核心任务不是预测文本内容, 而是判断那个文本与哪个图像是匹配的
## Method
### 直接做法的失败构建
- 作者直接将CNN作为图像识别器, 将Transformer作为文字预测器, 
- 他们发现这个方法极为低效, 一个模型为了精准的预测图片描述的每个词, 耗费了不必要的算力
### 转向对比学习
- 核心思想: 拉进相似的，推开不相似的。如此的零样本学习效率比之前的方法提高了4倍
- **对比学习流程**![[Pasted image 20251017144854.png]]
	- 在一个批次 (batch) 中，有 N 个图像和 N 个与之配对的文本。
	- **编码**：使用一个图像编码器 (Image Encoder)（如 ResNet 或 ViT）将 N 个图像编码成 N 个图像特征向量 $[I_1, I_2, ..., I_N]$。同时，使用一个文本编码器 (Text Encoder)（将 N 个文本编码成 N 个文本特征向量 $[T_1, T_2, ..., T_N]$。
	- **对比**：计算这 N 个图像特征和 N 个文本特征之间的所有 $N \times N$ 个可能的配对的余弦相似度，形成一个 $N \times N$ 的相似度矩阵。
	- **目标**：模型的目标是最大化对角线上 N 个正确配对 $(I_i, T_i)$ 的相似度，同时最小化其余 $N^2 - N$ 个错误配对 $(I_i, T_j)$ 的相似度。
- 最终完成了图片和文字领域的对齐
